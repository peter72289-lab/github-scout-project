How the Engine Actually Works

Transparent by Design

Analytical buyers — which is most of our operators — want to understand how intelligence is generated before trusting it enough to pay for it. So here's the exact methodology, not a marketing version of it.

Every number Scout surfaces is traceable to a source, a collection method, and a scoring formula. Nothing is a black box.

1
Source Collection (continuous)
Scout queries all 15 indexed platforms on rolling schedules — launch surfaces hourly, alternatives directories daily, maturity validators weekly. Each source has its own connector and refresh cadence.
2
Normalization (per result)
Every raw result — regardless of source — is normalized to the same 26-field schema: name, URL, source, category, license type, commit frequency, contributor count, star count, download count, and 17 more. Partial data is flagged, not hidden.
3
Deduplication (cross-source)
Fuzzy name matching + URL comparison catches the same project appearing across multiple sources. Merged records get a cross-source confidence boost proportional to how many sources corroborate them.
4
Scoring (six dimensions, 1–5 each)
Finishedness, Commercial Pain, Replacement Clarity, Angle Strength, Packaging Quality, Signal Density — each computed from measurable signals in the normalized data. Scores are visible on every result card. You see why a result ranked where it did.
5
Ranking (mode-specific rollup)
Exploratory Score = Finishedness + Angle + Packaging + Density. Savings Score = Pain + Replacement + Finishedness + Density. Mode detected from query language or selected manually. Max score: 20 per mode.
