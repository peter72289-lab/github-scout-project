Launch security baseline

Data Handling

The URL analyzer is designed to inspect public ecommerce storefront signals only. It does not log into Shopify admin, request passwords, request private API keys, or scan private network addresses.

Scanner Guardrails

  • Only HTTP and HTTPS URLs are accepted.
  • Localhost, private IP ranges, internal hostnames, and local network addresses are blocked.
  • DNS resolution is checked before fetching so public-looking hostnames cannot resolve to private network targets.
  • Redirects are limited and each redirect destination is validated before fetching.
  • Responses are size-limited and restricted to public HTML-like content.

What the Scan Reads

The scan reads public storefront HTML and known software signatures such as script URLs, widget markers, analytics tags, review tools, popup tools, and app embeds visible to a normal browser request.

What It Does Not Read

The scan does not access Shopify admin data, customer records, orders, private source code, hidden databases, or authenticated pages.

Operational Notes

Production recommendations should still be verified by the store owner before installing, removing, downgrading, or replacing any app that touches checkout, customer data, subscriptions, fulfillment, or compliance.
